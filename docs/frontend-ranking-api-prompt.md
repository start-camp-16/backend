# 프론트엔드 전달용 프롬프트 — 추천 랭킹과 전체 장소 목록

현재 프론트엔드의 구조, 기술 스택, 컴포넌트 패턴, 스타일을 먼저 확인한 뒤 아래 기능을 기존 화면에 자연스럽게 추가해 주세요. 기존 디자인을 전면 교체하지 말고 재사용 가능한 API 클라이언트, 타입, 로딩·오류 처리 패턴을 따르세요.

## 목표

장소 화면을 다음 두 용도로 분리합니다.

1. 사용자가 선택한 `서울 구 × 장소 카테고리`의 추천 장소를 최대 5개 보여주는 랭킹 영역
2. 추천 여부와 관계없이 전체 장소를 탐색하는 페이지네이션 목록 영역

백엔드의 기준 계약은 `shared/openapi.yaml`이며, 구현 완료 후 생성된 계약을 최종 기준으로 사용하세요.

## 사용할 API

### 메타데이터

- `GET /api/meta/districts`
- `GET /api/meta/categories`

구와 카테고리 선택지는 하드코딩하지 말고 위 API 응답으로 구성합니다.

### 추천 랭킹

```http
GET /api/rankings?district=강남구&category=관광지
```

- `district`, `category`는 필수입니다.
- 응답은 선택한 조합의 추천 장소를 1위부터 최대 5개까지 반환합니다.
- 해당 조합에 장소가 없으면 정상 응답으로 빈 `items` 배열이 옵니다.
- 페이지네이션은 없습니다.

예상 응답 형태:

```json
{
  "district": "강남구",
  "category": "관광지",
  "items": [
    {
      "rank": 1,
      "content_id": "123456",
      "category": "관광지",
      "title": "장소명",
      "address": "서울특별시 강남구 ...",
      "district": "강남구",
      "longitude": 127.0,
      "latitude": 37.5,
      "image_url": "https://...",
      "thumbnail_url": "https://...",
      "phone": null
    }
  ]
}
```

### 전체 장소 목록

```http
GET /api/locations?district=강남구&category=관광지&page=1&size=20
```

- `district`, `category`는 선택 조건입니다.
- `page` 기본값은 1입니다.
- `size` 기본값은 20, 최댓값은 100입니다.
- 필터가 없으면 전체 장소를 반환합니다.

예상 응답 형태:

```json
{
  "items": [
    {
      "content_id": "123456",
      "category": "관광지",
      "title": "장소명",
      "address": "서울특별시 강남구 ...",
      "district": "강남구",
      "longitude": 127.0,
      "latitude": 37.5,
      "image_url": "https://...",
      "thumbnail_url": "https://...",
      "phone": null
    }
  ],
  "pagination": {
    "page": 1,
    "size": 20,
    "total_items": 6518,
    "total_pages": 326
  }
}
```

## 화면 동작

### 추천 랭킹 영역

- 구와 카테고리가 모두 선택되었을 때 `/api/rankings`를 호출합니다.
- 각 카드에 1~5위 배지를 명확히 표시합니다.
- 장소명, 카테고리, 주소, 대표 이미지를 우선 노출합니다.
- 이미지가 없으면 레이아웃이 무너지지 않는 공통 대체 이미지를 사용합니다.
- 결과가 없으면 오류처럼 보이지 않도록 `해당 조건의 추천 장소가 없습니다`라는 빈 상태를 표시합니다.

### 전체 장소 영역

- 구와 카테고리 필터를 각각 선택 또는 해제할 수 있어야 합니다.
- 필터가 변경되면 페이지를 1로 초기화합니다.
- 서버의 `pagination` 값을 사용해 이전·다음 또는 페이지 이동 UI를 구성합니다.
- 추천 랭킹과 전체 목록의 로딩·오류 상태를 서로 독립적으로 관리합니다.

## 구현 조건

- API 호출 파라미터와 응답 타입을 명시적으로 정의합니다.
- API의 `snake_case` 필드는 기존 프로젝트 변환 규칙이 있을 때만 변환합니다.
- 요청 중 중복 호출이나 이전 요청의 늦은 응답으로 화면이 덮이는 문제를 방지합니다.
- 모바일과 데스크톱에서 필터와 카드가 읽기 쉽게 보여야 합니다.
- 정상, 로딩, 빈 결과, API 오류, 페이지 이동을 테스트합니다.
- 백엔드가 아직 배포되지 않았다면 API 계층만 임시 mock으로 대체하되, 컴포넌트 내부에 mock 데이터를 직접 넣지 마세요.
